# IP Service

Небольшой сервис на FastAPI, который возвращает IP-адрес клиента и, при
необходимости, временно сохраняет связь имени компьютера с этим IP.

Проект запускается одной командой через Docker Compose. В составе есть:

- FastAPI-приложение;
- nginx как обратный прокси;
- Redis для хранения временных привязок;
- тесты для основной логики.

## Как запустить

```bash
docker compose up --build
```

После запуска сервис доступен по адресу:

```text
http://localhost:8000
```

## Как работает API

Обычный запрос возвращает IP клиента:

```bash
curl http://localhost:8000/
```

Пример ответа:

```json
{
  "ip": "172.20.0.1",
  "computer_name": null,
  "saved": false
}
```

Если передать имя компьютера, сервис сохранит привязку имени к IP в Redis:

```bash
curl "http://localhost:8000/?computer_name=my-pc"
```

Пример ответа:

```json
{
  "ip": "172.20.0.1",
  "computer_name": "my-pc",
  "saved": true
}
```

Поле `saved` показывает, была ли сохранена привязка. Если имя компьютера не
передано, сохранять нечего, поэтому возвращается `saved: false`.

## Хранение в Redis

Привязка сохраняется в Redis с TTL:

```text
computer:my-pc -> 172.20.0.1
```

Время жизни задается переменной `BINDING_TTL_SECONDS`. По умолчанию используется
`300` секунд.

Проверить сохраненное значение можно так:

```bash
docker compose exec redis redis-cli GET computer:my-pc
docker compose exec redis redis-cli TTL computer:my-pc
```

## Определение IP за прокси

Запросы проходят через nginx, который передает в FastAPI прокси-заголовки.
Сервис проверяет IP в таком порядке:

1. `Forwarded`
2. `X-Forwarded-For`
3. `X-Real-IP`
4. адрес клиента из запроса

Локально при запуске через Docker в ответе может быть адрес вида `172.20.0.1`.
Это нормальное поведение: такой IP относится к внутренней Docker-сети. Логику
работы за прокси можно проверить так:

```bash
curl -H "X-Forwarded-For: 8.8.8.8" http://localhost:8000/
```

Ожидаемый ответ будет содержать:

```json
{
  "ip": "8.8.8.8",
  "computer_name": null,
  "saved": false
}
```

## Имя компьютера через заголовок

Браузер не передает реальное имя компьютера пользователя. Поэтому основной
способ указать имя компьютера - query-параметр `computer_name`.

Дополнительно сервис поддерживает заголовок `X-Computer-Name`. Это удобно, если
имя компьютера добавляет доверенный прокси или внутренний шлюз:

```bash
curl -H "X-Computer-Name: my-pc" http://localhost:8000/
```

Если одновременно переданы query-параметр и заголовок, приоритет остается за
query-параметром.

## Проверка работоспособности

Endpoint для проверки состояния:

```bash
curl http://localhost:8000/healthz
```

Ожидаемый ответ:

```json
{
  "status": "ok"
}
```

## Тесты

```bash
pytest
```

Тесты покрывают:

- определение IP по прокси-заголовкам;
- сохранение привязки `computer_name -> IP`;
- TTL для записей в Redis;
- основной endpoint;
- endpoint проверки состояния.
